大家好，我的新书《Flutter开发之旅从南到北》已经在各大书店开始同步发售了，如书名表达的意义，这是一本 “从入门到进阶，从理论到实践” ，“从 Dart 到 Flutter，从源码层到应用层” 的书，其中不乏介绍了一些其他作者鲜少提及的内容，包括框架层中的三棵树、状态管理方案、布局约束等知识点。

我要写的这一系列文章旨在对书中没有深入分析的部分再做补充，以便读者能够在学有余力的情况下继续夯实自己的基础。今天要讨论的话题是 Flutter 中的状态，虽然书中已经专门拿出来一章介绍这部分的内容了（1），本篇文章要做的就是通过实例带大家更深刻的体验其中的奥妙。

那么，你们通常是怎么改变 StatefulWidget 中的状态的？

setState？这个方式最被我知道，每个 StatefulWidget 都可以在自己的状态对象中调用 setState 这个生命周期方法实现自己状态的更新：

这时，setState 方法会调用 Element.markNeedsBuild()方法，随即将该 Widget 对应的 Element 对象标记为脏（dirty）状态来表示它内部数据已经改变了，然后就会触发 build 重建出新的 Widget 对象。
